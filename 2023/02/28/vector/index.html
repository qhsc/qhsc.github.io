150. 逆波兰表达式求值

  • 难度： 2
  • 题目：给定一个字符串数组 tokens，表示一个根据逆波兰表达式表示的算术表达式，求表达式的值是多少。注意：答案和所有的中间结果都可以用 int32 表示，除法总是向 0 截断，且不会除以 0。
  • 例子：输入：tokens = ["2","1","+","3","*"]；输出：9；解释：该算式转化为常见的中缀算术表达式为：((2 + 1) * 3) = 9
  • 解析：逆波兰表示法是先有操作数，再有操作符，遇到操作符需要 pop 出前两个数字作为操作数。

155. 最小栈

  • 难度： 2.5
  • 题目：实现一个最小栈，相比普通的栈额外新增一个 getMin 方法，能得到当前栈内所有元素的最小值，要求在常数时间内能完成该操作。
  • 解析：需要能在常数时间内完成获取最小值的操作，最好是最小值就是已经存储好的。考虑栈的特性是一个个元素添加的，假设已经已知当前所有元素的最小值，则再添加一个元素的最小值也是很容易计算的。所以可以把每添加一个元素后集合最小值存成一个栈，和元素栈一起 push 和 pop。

224. 基本计算器

  • 难度：4
  • 题目：实现一个支持加、减法、括号优先级的计算器。输入是一个表达式，可能含有空格，并且可以把 - 号当作一元运算符号。
  • 解析：首先整理表达式，去掉空格，并且在左右两端添加括号，并把 - 一元运算符转化成二元运算符。
    • 遇到数字就push number
    • 遇到 ( 就push operator，遇到 ) 就 pop operator 进行计算。这里一定要尽量能把该算的都算了。
    • 遇到其他操作符，计算当前操作符的优先级是否高于之前的，如果高于则 push 进去，如果不高，则先把之前的运算符给算了，同样能算的都算了。

331. 验证二叉树的前序序列化

  • 难度：3

  • 题意：给定一个字符串表示一颗二叉树的前序遍历，非空节点是一个整数值，空节点的值是一个 #，节点之间用 , 分割，需要你验证这样的一个序列是否是一颗合法的二叉树的前序遍历。

  • 解析：分析可知，这颗树中除了空节点以外，每个节点都挂了两个子节点，而空节点不挂任何的子节点。我们可以想象每处理一个元素都是把这个节点挂到某个节点下面，如果这个元素没有节点可挂，则说明这个序列是有问题的。我们对每个节点挂了两个元素后，这个节点就不能再挂了，需要删掉。对于根节点可以假设它还有一个父节点，但是这个父节点只能挂一个子节点。用栈来存储这些能挂载子节点的节点。遍历完成后，这个数组应该是空的。

  • 优化算法：也可以这样考虑，我们有一个能挂载节点的容量 cap

    • 每来一个节点，池子容量 -1，如果不能减少，说明有问题。
    • 同时，如果是遇到非空节点则池子容量 +2。

    遍历完成后，池子容量应该为 0。 显然为了挂载根节点初始化池子的容量应该为 1。这也推导出一个式子 cap = 0 = -节点数 + 2*非叶子点数 + 1。即 叶子节点数 = 非叶子节点数 + 1。

402. 移掉k位数字

  • 难度：3
  • 题目：给定一个数字，求删除其中的 k 位数之后，剩下的最小数字是多少？数字是10进制的，并且用字符串表示。返回值也是用字符串表示，但是不能含有前导0。
  • 解析：如果删除一个位数之后能使得数字变小，那么意味这在这位数之后一定存在这某个数比他小。也就是说，新来一位数，我们总是能把它之前比它大的数删掉（如果合法的话），但是它自己我们还不能决定是否删除，得等到下一轮才能判断。
  • 注意：前导0的问题不要过早考虑，否则容易影响思路。

456. 132 模式

  • 难度：4

  • 题目：给定一个整数数组，判断大小是否存在 132 模式的子序列。即判断是否存在 i < j < k 且 nums[j] > nums[k] > nums[i]。

  • 解析：对于任意符合要求的 i 则右边一定存在一个递减的包含两个元素的递减子序列，如果我们能算出这些可能的序列中最小的 nums[k] 是多少，则如果满足 nums[i] < nums[k] 就存在一个符合题目要求的序列。现在考虑如何计算最小的 nums[k]。考虑一个构建单调递减栈的过程，每次来一个新元素，都会检查栈顶元素是否比当前元素小，如果是则会弹出该元素，并且会重复此过程，当栈顶元素比它大时则会停止，并将该元素加到栈顶。我们发现有几个重要性质：

    • 单调递减栈的元素大小是递减的
    • 其栈顶总是当前元素
    • 任何元素都会进栈
    • 每增加一个元素前，弹出的元素的最后一个就是序列中小于当前元素的最大元素
    • 再考虑构建过程经过了多轮的情况，能发现上述的最大元素是递增的

    利用最后两个性质，我们可以从右往左构建单调栈，并算出多轮构建时 nums[k]的最大值。

  • 难度：3
  • 题目：给定一个环形数组 nums，判断其中每一个元素的下一个比它大的元素是多少，如果没有的话，则该值认为是 -1。
  • 解析：对于元素 i, 假设我们把它右边的所有元素按照从右至左的顺序构建一个单调递减栈，则显然能发现此时栈顶的元素是 nums[i+1]， 栈顶的下一个元素刚好是大于 nums[i+1] 的下一个可能的元素。在构建单调栈的过程中，我们总是会把小于当前元素的元素出栈，所有此时栈顶的元素就刚好是大于当前元素的 下一个可能元素，得解。注意由于是循环数组，需要事先建立一个单调栈。
