1049. 最后一块石头的重量 II

  • 难度：3
  • 题目：给定一堆石头，每个石头有重量 stones[i]。执行以下步骤直到只剩一个石头：任选两个石头进行碰撞相消。如果重量相等，则碰撞以后不会留下任何石头；如果重量不等， 两块石头碰撞以后会留下一块石头，重量为两块石头重量的差值。问：留下的石头的重量最小可能是多少？
  • 解析：
    • 分析：每次碰撞都会消去2的整数倍的重量，则最终被消去的重量是 2*x，这里 x 是正整数，还留下 y 重量的石头。则有 sum(stones)=2*x+y。因此，如果我们能把石头分作两堆，并且两堆的差距是最小的，则此时的差距就是 y ，也就是题目所求。
    • 解法：上述问题可以转化为：从一堆数字中选取若干个，其能得到的和当中不大于且最接近这一堆数的和的一半的数是多少。这是一个 0-1 背包问题。用动态规划求解。

1105. 填充书架

  • 难度：3
  • 题目：你现在有一个书架宽度 width 是固定的，层数是可以拓展的，然后还有一堆书，这堆书已经放好了顺序，每本书都有自己的厚度 thickness_i 和 高度 height_i。你需要按照顺序将这些书放到书架里面，排放的方式就像图书馆的书一样，即：每一层的书籍都是竖起来的，它们的厚度加起来不能超过书架的宽度，然后每一层的宽度由这一层书当中最高的决定，一层书可以不放满。求最后的书架的最小高度是多少？
  • 解析：采用典型的动态规划算法，假设 miniHeight[i] 代表存放第 i 本书后书架的最低高度，当新加一本书时，如何推导第 i+1 本书所需的最低高度呢？新加的这一本书一定属于最后一层，则书架的高度一定等于上一层的高度加上本层的高度。我们假设层高为 height，则有 miniHeight[i+1]=miniHeight[x]+height && sum(thickness[x+1,i+2]) <= width && x >= -1 && x <= i。

920. 播放列表的数量 [HARD]

  • 难度：4
  • 题目：你的播放器里面有 N 首歌，你的对象在旅途中需要一个长度为 L 的播放歌单，这个歌单里面的歌可以重复。但是得满足以下两个规则：1. 每首歌必须至少播放过一次；2. 一首歌两次播放的最短间隔为 K。求问可能的播放列表数量（结果对于 1e9+7 的余数）。
  • 提示：0 <= K < N <= L <= 100
  • 解析：毫无技巧，典型的动态规划难题。感觉 K 的限制和至少播放一次的限制应该是用来限制状态转移方程的，不是变量。
    1. 首先来尝试定义状态，假设 listNum[l][n] 代表在长度为 l 的播放列表里面有 n 首不同的歌的可能的播放列表个数，则 listNum[N][N] 即为所求。显然 list[0][0]=1。
    2. 归纳推导状态转移方程。list[l][n] 增加一首歌之后可能会得到 list[l+1][n] 或者是 list[l+1][n+1]，分别对应增加一首已经出现过的歌和一首没出现过的歌。
      • 增加一首已经出现过的歌的可能选择有 max(0,n-K) 种，但是要满足 l>K。
      • 增加一首未出现过的歌的可能选择有 max(0, N-n) 种。

474. 一和零

  • 难度：3
  • 题意：给一个二进制字符串的数组 strList，比如 {"01","00","1","111"}，要你从中选一个子集出来，子集中这些字符串的 0 和 1 的个数分别不能超过 N 和 N。问最大的子集长度是多少。
  • 解析：这个其实就是在满足限制条件的情况下如何选择字符串的问题，是二维的 0-1 背包问题。设 maxPossible[m][n] 为尽量填满 0 和 1 个数分别为 m 和 n 时，集合的最大可能长度。遍历字符串数组，对于第 i 个字符串，0 和 1 的个数分别是 a 和 b ，从大到小的方向遍历 maxPossible，则有 maxPossible[m][n] = max(maxPossible[m-a][n-b]+1, maxPossible[m][n])
  • Takeaway：0-1 背包问题的外层循环一般是对需要选择的物品做遍历，内层更新动态规划的临时变量。

435. 无重叠区间

  • 难度：3
  • 给定一个区间的集合 intervals，其中 intervals[i] = [start_i, end_i]。从中删掉0个或多个区间，使得剩下的区间之间没有重叠，求至少需要删除多少个区间。
  • 解析：直观的感觉有点像0-1 背包：从当中选取一个最大的子集，子集区间之间没有重叠。这类问题一般用动态规划求解。
    • 解法一：首先尝试定义状态变量，设不大于第 i 个区间的右端点的区间集合中满足不重叠要求的最大集合为 maxCap[i]。接下来考虑如何更新的问题，首先直观的想法是最好 intervals 中给出的区间是按照区间右端点排好序的，只有这样按序处理更新状态才是对的。显然有 maxCap[i] = max(maxCap[i], maxCap[j]+1), 且满足 j >= 0 && j<i && start_i >= end_j。二重遍历 i,j 更新即可。
    • 解法二：这个解法是在解法一的基础上通过推导归纳出来的，思想是一样的，只是通过归纳法将遍历过程 O(n*n) 的复杂度降低为 O(n)，看起来其实就是个贪心算法，但是需要证明正确性。假设 maxCap[i] 对应的集合的区间的最右端点为 edge[i]。假设 x<y<i，x+1=y，
      • 显然如果 maxCap[y] > maxCap[x]，不用遍历 x。
      • maxCap[y] == maxCap[x]时，如果 interval[i] 的左端点大于等于 edge[y] 时，也不用再去遍历 x 了。
      • 唯一需要去遍历 x 的情况就是，edge[x] <= start_i < edge[y] && maxCap[x]==maxCap[y]。我们不想遍历 x，所以要打破这种条件，也就是说如果我们发现 y 相对于 x 集合数量没有增长时，不要推进 edge[y]，虽然 edge[y] 不是唯一的，也可以推进，但是不推进却可以减少我们的计算量。推广一下，此时我们就只需要对比 edge[i-1] 和 start[i] 的关系就行了，edge 数组单调增，可简化成一个数值。

416. 分割等和子集

  • 难度：2
  • 题意：给定一个正整数集合 nums，问能否将这个集合拆成两个，并且这两个集合中元素得和相等。
  • 解析：0-1 背包问题，遍历元素顺序无关紧要。设状态变量 maxValidSumAt[i] 表示不大于 i 的最大的子集和。题目所求即为：maxValidSumAt[sum/2]。更新方程，maxValidSumAt[i+num]=maxValidSumAt[i]+num，如果从小到大的方向更新，则状态遍历会被覆盖写，反过来从大到小的方向更新即可。

343. 整数拆分

  • 难度：2
  • 题目：给定一个正整数 n, 2 <= n <= 58，将其拆分成为 k, k>=2 个整数的和，求这些整数的最大的可能积是多少？
  • 解析：不是数学题。用动态规划做，设 product[i] 代表 i 拆分后的最大可能乘积。则有 product[i] = max(product[x]*product[y]), x+y=i，但是这个式子是错的，因为仅仅代表了至少拆成2+2=4个部分的情况，还因该有拆分2、3部分的情况，即 product[x]*y, x*product[y], x*y，合并在一起即是：product[i] = max(product[i], max(x, product[x]) * max(y, product[y])), x+y=i, 1<=x<=i/2。

322. 零钱兑换

  • 题意：假设有不同面值的硬币若干，每种面值个数无限，面值的可能存于 coins 数组中；给定一个整数 amount 表示总金额，求组成这个总金额最少需要多少个硬币，不能组成时返回 -1 。

309. 最佳买卖股票时机含有冷冻期

  • 难度：3

  • 题意：给定一个数组 prices 代表某个股票每天的价格，你可以在第 i 天买入，然后在第 j,j>i 天卖出，下次买入的时机必须要冷却一天，即下次只能在第 j+2 天或者以后才能买入。注：在买入前必须手上没有股票。求最大利润。

  • 解析：每天有四种状态，分别是：无持有可买入、持有、卖出、无持有不可买卖。设这四种状态对应的利润分别为 x,y,z,w。则状态转移方程有

    • x = max(x,z,w)
    • y = max(y,x-price[i])
    • z = y+price[i]
    • w = z

    初始状态为 x=0,y=-prices[0],z=0,w=0

198.打家劫舍

  • 难度：2
  • 题意：你是一个小偷，沿街有一排房屋，屋里有报警系统，如果你进入了相邻的两间房屋则就会触发报警。求能偷取的最多钱财是多少。
  • 解析：很像309题，但更简单。经过每间屋子前只可能有两种状态：偷和不偷。假设两种状态下能获得的钱财总和分别是 steal 和 skip，则有 steal[i+1] = skip[i]+money[i]，skip[i+1] = max(skip[i], steal[i])。

70.爬楼梯

  • 难度：1
  • 题目：假设你正在爬楼梯，一共有 n 阶，每次你可以爬 1 级或者 2 级台阶。问一共有多少种方法达到顶层。
  • 解析：假设 ans[i] 代表第 i 级阶梯的可能方法，则 ans[i+2] = ans[i+1] + ans[i]。因为只有三个变量，可以稍微优化下内存占用。

1144. 递减元素使数组呈锯齿状

  • 难度：2
  • 题意：给你一个整数数组 nums，每次操作可以将一个元素 -1，问要使得数组呈现锯齿状，最少的操作此时是多少？一个锯齿数组是指奇数位置的数字都大于相邻位置的数字，或者都大于相邻位置的数字。注：只能 -1
  • 解析：不管是哪种锯齿数组其实每个元素都只有两种状态，要么是峰，要么是谷。设 ridge, valley 分别代表该元素是峰或者谷时的状态，则有
    • ridge.num = num
    • ridge.ops = valley.ops + max(0, valley.num+1-num)
    • valley.num = min(ridge.num-1, num)
    • valley.ops = ridge.ops + max(0, num-ridge.num+1)
    • 答案： min(ridge.ops, valley.ops)
